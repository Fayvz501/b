# 💰 Credit Calculator VK Bot

VK-бот: расчёт кредита по двум схемам (аннуитет / дифференцированный),
графики структуры платежа, сравнение остатка долга, PDF-отчёт с полным
графиком на весь срок.

## 🚀 Возможности

- **Финансовая математика:** точный расчёт ежемесячного платежа,
  переплаты, остатка долга по обеим схемам.
- **4 графика на matplotlib:** структура платежа (аннуитет / дифф.),
  остаток долга, сравнение размеров платежа.
- **Банковский PDF-отчёт** (ReportLab): сводная таблица + полный график
  платежей помесячно.
- **Callback-кнопки VK API:** переключение между графиками без перезапуска
  диалога — нажал и сразу получил картинку в чат.
- **Исследовательская часть:** математическое доказательство, что
  дифференцированная схема всегда даёт меньшую переплату при `i > 0`.

## 🛠️ Деплой на Render через GitHub

### 1. Создание группы ВК

1. https://vk.com/groups → создать сообщество.
2. **Управление → Сообщения** → включить.
3. **Управление → Работа с API → Ключи доступа** → создать токен с правами
   *Сообщения сообщества*, *Управление*, *Фото*, *Документы*. Скопируй —
   это `VK_TOKEN`.
4. **Работа с API → Long Poll API** → включить, версия **5.199** (или
   максимальная доступная), отметить события:
   - `Входящее сообщение`
   - `Callback-кнопки: ответ`
5. **Работа с API → Callback API → подтверждение** не нужно — мы
   используем **Long Poll**, не Callback API.
6. ID сообщества (цифры в URL `vk.com/club...`) — это `VK_GROUP_ID`.

### 2. Залить код на GitHub

```bash
cd credit-bot
git init
git add .
git commit -m "init: credit calculator vk bot"
git branch -M main
git remote add origin https://github.com/<твой-логин>/credit-bot.git
git push -u origin main
```

### 3. Деплой на Render

1. https://dashboard.render.com → **New → Blueprint**.
2. Подключи репозиторий → Render найдёт `render.yaml` сам.
3. Когда попросит, заполни секреты:
   - `VK_TOKEN` — токен из шага 1.
   - `VK_GROUP_ID` — числовой ID сообщества.
4. **Apply** → Render начнёт сборку (~3 минуты).
5. После статуса *Live* открой URL вида
   `https://credit-bot-xxxx.onrender.com/health` — должен вернуться
   `{"status": "ok"}`.
6. Иди в свою группу ВК → напиши «Начать» → бот ответит.

### 4. Особенности free-tier Render

- Диск **эфемерный** — `state.db` живёт в `/tmp` и обнуляется при
  перезапуске. Это ок: состояния короткие (3 шага диалога).
- Сервис **засыпает** после 15 минут без HTTP-запросов. Long Poll держит
  исходящее соединение, но Render всё равно может усыпить процесс.
  Решение: настрой **UptimeRobot** (бесплатно) на пинг
  `https://<your-app>.onrender.com/health` каждые 5 минут — бот всегда
  будет онлайн.

## 💻 Локальный запуск

```bash
cp .env.example .env
# отредактируй .env, вписав VK_TOKEN и VK_GROUP_ID
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
export $(grep -v '^#' .env | xargs)
python bot.py
```

Открой `http://localhost:10000/health` — там должно быть `ok`. Параллельно
бот слушает Long Poll и отвечает в группе ВК.

## 🧮 Математическое доказательство

Кратко: переплата по любой схеме равна `i · Σ B_k`, где `B_k` —
остаток долга после k-го месяца. Дифференцированная схема даёт линейно
убывающий остаток `B_k = S(n-k)/n`, аннуитетная — медленнее
(в начале платежа большая часть уходит на проценты). Поэтому
`Σ B_k^{ann} > Σ B_k^{diff}`, и `I_{ann} > I_{diff}` строго при `i > 0`.

Точные формулы:

- **Дифф.:** `I_d = i · S · (n+1) / 2`
- **Анн.:** `I_a = n · S · i(1+i)^n / ((1+i)^n − 1) − S`

Численная разница на `S = 1 000 000 ₽`:

| ставка \ срок | 12 мес | 60 мес | 120 мес | 240 мес |
|---|---|---|---|---|
| 5%   | +0.4% | +1.7% | +3.3%  | +6.5%  |
| 10%  | +0.8% | +3.4% | +6.6%  | +12.7% |
| 15%  | +1.2% | +5.1% | +9.7%  | +18.5% |
| 20%  | +1.6% | +6.7% | +12.7% | +23.7% |

При `i·n → ∞` отношение `I_a / I_d → 2`.
